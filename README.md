Практика 21 — Redis кэширование
Что сделано
Добавлено кэширование запросов через Redis.

Закэшированные маршруты
Маршрут	Время кэша
GET /api/users	1 минута
GET /api/users/:id	1 минута
GET /api/products	10 минут
GET /api/products/:id	10 минут

Как работает кэш
1. Первый запрос → данные из базы → сохраняются в Redis

2. Второй запрос → данные из Redis (быстро)

Как запустить
cd 21
docker run -d --name redis-cache -p 6379:6379 redis
node server.js

Практика 22 — Балансировка нагрузки
Что сделано
Настроен Nginx как балансировщик между тремя серверами.

Архитектура
        Запросы
           ↓
    Nginx (порт 8080)
      /     |     \
     ↓      ↓      ↓
 Сервер1  Сервер2  Сервер3
 (3000)   (3001)   (3002) backup

 Алгоритм балансировки
Round-robin (по кругу)

При отказе сервера Nginx перестаёт отправлять на него запросы

Резервный сервер (3002) включается, если основные не работают

Как запустить
cd 22

# Терминал 1
PORT=3000 node server.js

# Терминал 2
PORT=3001 node server.js

# Терминал 3
PORT=3002 node server.js

# Терминал 4
sudo nginx -c $(pwd)/nginx.conf

Отказоустойчивость
1. Остановить сервер на порту 3000

2. Запросы идут только на 3001

3. Остановить 3001 → запросы идут на резервный 3002

Альтернативный балансировщик (HAProxy)
haproxy -f haproxy.cfg
curl http://localhost:8081/

## Практика 23 — Docker и контейнеризация

### Что такое Docker
Docker — программа для запуска приложений в изолированных контейнерах.

### Что такое контейнеризация
Технология упаковки приложения со всеми зависимостями в «контейнер», который работает одинаково на любом компьютере.

### Плюсы контейнеризации
1. **Переносимость** — работает везде (Mac, Linux, Windows, облако)
2. **Изоляция** — контейнеры не мешают друг другу
3. **Лёгкость** — запуск за секунды, мало места
4. **Воспроизводимость** — одинаковая среда у всех разработчиков
5. **Масштабирование** — легко запустить много копий

### Команды Docker, которые использовались

```bash
# Запустить Redis
docker run -d --name redis-cache -p 6379:6379 redis

# Запустить MongoDB
docker run -d --name mongodb -p 27017:27017 mongo:latest

# Остановить контейнер
docker stop redis-cache

# Запустить остановленный контейнер
docker start redis-cache

# Удалить контейнер
docker rm redis-cache

# Посмотреть запущенные контейнеры
docker ps
